## Rules of thumb

1. For each gists, create a new folder
2. For each folder, make sure to use environments to ensure reproducible workflows as much as possible
    * For python, use [uv](https://docs.astral.sh/uv/)
    * For bonsai use [environments](https://bonsai-rx.org/docs/articles/environments.html)
3. It is advised to add a readme to each gist
4. If possible add MINIMAL data (or mocks) to reproduce the gists
5. Do NOT add full datasets for tests/examples. Small files are fine (e.g.: keep it to <10KB per gist). If you need to share larger datasets, consider using external storage solutions and provide download links in the readme files or scripts.
